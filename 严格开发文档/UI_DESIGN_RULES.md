# UI_DESIGN_RULES.md

> 说明：本文件定义 titanLX 当前阶段的**产品形态规则**。  
> 当前阶段只允许一种合法实现：**默认卡片态 + 节点主体外部下方独立编辑区**。  
> 不允许继续实现成“大表单节点”。

## 视觉目标
整体是 **媒体卡片式节点画布**，不是工程参数表，不是长表单节点集合。

要求：
- 深色背景
- 卡片式节点
- 节点圆角 20-24px
- 节点默认态应是紧凑卡片
- 节点编辑区应是下方独立展开的次级层
- 右侧面板是辅助，不是主工作区

---

## 当前阶段唯一合法节点结构
### 第一层：默认卡片态（必须常驻）
默认只能看到紧凑节点卡片，至少包含：
1. Header：图标、标题、状态、删除
2. Summary：摘要文本 / 缩略预览 / 当前结果缩略图（如有）
3. Quick Actions：少量轻量动作（例如：输出按钮 + 单一主动作）

### 默认卡片态禁止展示的内容
- 完整参数区
- 长 textarea / 完整 prompt 编辑器
- 全量 customPrompting
- 全量 template 配置
- 摄像机完整表单
- 长的 runtime 明细区
- 过多 Quick Actions 堆成工具栏

### 第二层：节点主体外部下方独立编辑区（按需展开）
点击节点缩略卡片后，在节点主体**外部下方独立展开**编辑区。编辑区至少包含：
1. Prompt / 文本输入区
2. 参数区（模型、比例、尺寸等）
3. 上游输入展示区
4. 编译提示词预览区
5. 结果明细区 / 运行时状态区

### 编辑区红线
- 不能只是节点主体内部继续向下拉长
- 不能做成 popup / drawer / 右侧替代面板 作为当前阶段主实现
- 必须肉眼一眼看出“默认卡片态”和“编辑区”是两层结构
- 必须明确看出编辑区是节点主体外部的次级层

---

## TextNode / ImageNode 当前阶段具体规则
### TextNode 默认态
允许：
- 标题
- 状态 badge
- 文本摘要
- 少量动作入口

禁止：
- 完整 textarea 默认展开
- customPrompting 全量字段默认展开
- 上游/下游大段调试信息默认展开

### ImageNode 默认态
允许：
- 标题
- 状态 badge
- 结果小图 / 占位缩略图
- 模型/比例的轻量摘要
- 少量动作入口

禁止：
- 模型/比例/尺寸完整表单默认展开
- prompt textarea 默认展开
- camera 全字段默认展开
- 完整 template/runtime 区块默认展开

---

## 关键交互要求
- 用户可以手动拖拽节点
- 用户可以手动拖拽 handle 连线
- 双击边或显式按钮删除连线
- Delete 删除节点或边
- 节点内表单控件必须 `nodrag`
- 点击节点缩略卡片才展开编辑区
- 点击画布空白处关闭编辑区
- 默认不允许多个编辑区同时展开，除非后续规范明确放开

---

## 节点互联要求（当前阶段硬规则）
以下连接方向必须真实可用：
- Text -> Image
- Image -> Image
- Image -> Text

“真实可用”至少包括：
1. 能创建边
2. 能删除边
3. 下游 upstream 立即更新
4. 编辑区中能看到上游数据
5. 删除边后 upstream 同步消失

不能只因为代码里存在 handle / connectionRules 就判完成。

---

## 输出创建链要求
- 点击输出按钮/输出锚点，弹出创建菜单
- 选择类型后在附近创建下游节点
- 自动建边
- 下游节点收到 upstream 数据
- 点击输出按钮创建下游，与拖拽到空白处创建下游，必须共用同一动作路径

---

## 运行时结果显示硬规则
Image Generate 当前阶段的最低可交付标准不是“状态变化”，而是：
1. 点击 Generate
2. 发起真实请求
3. 返回真实图片
4. 节点中显示真实返回图片

如果只有 success/error 状态变化，但节点中没有显示真实返回图片，视为未完成。

---

## Generate 结果反作弊要求
以下都不算真实结果图：
- picsum / placeholder 图
- 本地 mock 图
- 固定测试图
- 与本次请求无关的旧图

验证时应优先证明：
- 这是本次真实请求返回的图
- 节点中确实显示了它
- 失败时节点中确实显示了错误
