# titanX 全节点能力与链接总表

记录时间：2026-03-11
状态：基于用户逐图确认 + 官方 API 资料整合后的第一版正式收口稿

---

## 1. 整体产品判断

当前 titanX 的节点系统不是一堆孤立小卡片，而是一个由复合节点组成的多模态工作流画布。

每类节点都不是单一输入框，而是至少包含：
- 节点头部/类型层
- 中间内容显示层
- 下方编辑执行层

对于生成型节点，统一应按以下逻辑理解：
- 节点可先空态创建
- 有内容后进入编辑态/结果态
- 结果可作为下游输入
- 多结果时统一按 `primaryResult + secondaryResults[]` 理解

---

## 2. 统一画布规则

### 2.1 节点创建
当前已确认支持：
- 左侧栏新建节点
- 双击画布空白处新建节点
- 从节点输出口拖拽创建下游节点
- 左侧底部 `上传` 入口按文件类型自适应创建节点

`上传` 当前至少支持：
- 图片
- 视频
- 音频
- 文本 / TXT

### 2.2 节点展开规则
统一按三段式理解：
- 闭合态：主要显示中间内容块
- 展开态：上标题区 + 中显示区 + 下编辑区
- 文本框放大只是编辑区放大，不改变下方按钮逻辑

### 2.3 画布交互
当前已确认：
- 拖动框选多个节点
- 右键拖动画布
- 空白处右键菜单：上传 / 添加资产 / 添加节点 / 撤消 / 重做 / 粘贴
- 单节点右键菜单：编辑 / 创建资产 / 复制 / 粘贴 / 副本 / 删除 / 复制到剪贴板
- 多选后支持创建资产、添加到现有资产
- 多节点支持打组、解组、整组执行、创建模板、整组移动

## 3. Text 节点

## 3.1 定位
Text 节点不是普通文本框，而是：
- 结构化提示词编辑器
- 文本分析节点
- 上游内容汇总节点
- 文本结果写回节点

## 3.2 上游输入
当前已确认左侧输入只接受：
- 文本节点
- 图片节点

其中图片输入到 Text 节点时，本质更接近：
- 把图片作为大模型附件 / 参考图挂载到当前 Text 节点
- 不是自动生成结构化下游 payload

## 3.3 分析执行规则
当前已确认：
- 上游内容接入后不会自动分析
- 必须由用户在底部输入框手动输入任务词，例如 `分析`
- 执行后，模型基于当前接入的所有上游文本/图片进行分析
- 分析结果写回当前 Text 节点中间正文区
- 每执行一次，默认刷新正文区内容，而不是叠加旧结果

## 3.4 对下游的写入
当前产品真实逻辑不是自动把分析结果沿 edge 传下去，而是：
- 先在 Text 节点内点击 `复制全部`
- 再手动粘贴到下游节点文本框

已经确认的新交互定义：
- 按钮名：`插入到下层开头`
- 行为：把当前 Text 节点内容插入到已连接下游节点文本框开头
- 当前适用下游：
  - Text
  - Image
  - Video
  - Audio

## 3.5 顶部工具条定位
Text 节点顶部工具条当前不应按普通富文本排版器理解，而应按“节点外观 + 结构化提示词注入 + 编辑辅助 + 跨节点动作”四层理解。

当前已确认 13 个按钮，顺序为：
1. 颜色
2. H1
3. H2
4. H3
5. 正文/段落
6. 粗体
7. 斜体
8. 无序列表
9. 有序列表
10. 分割线
11. 复制全部
12. 全屏
13. 插入到下层开头（产品增强按钮）

其中分类规则已确认：
- 节点外观样式：`颜色`
- 提示词结构注入：`H1 / H2 / H3 / 正文/段落 / 粗体 / 斜体 / 无序列表 / 有序列表 / 分割线`
- 编辑辅助：`复制全部 / 全屏`
- 跨节点动作：`插入到下层开头`

特别说明：
- `颜色` 只改变当前 Text 节点自身颜色，不注入提示词
- `复制全部 / 全屏 / 插入到下层开头` 都不是提示词注入按钮
- 真正进入 prompt 编译链的是 `H1` 到 `分割线` 这一组结构化语义按钮

## 4. Image 节点

## 4.1 定位
Image 节点是：
- 图片生成节点
- 图片编辑节点
- 图片结果分发节点

## 4.2 状态
当前已确认至少包括：
- 空态
- 已载入图片态
- 通用图片编辑态
- 各编辑子模式
- 多结果态

## 4.3 上传后的结构
上传图片后：
- 节点进入素材/结果态
- 通常收掉左输入口
- 保留右输出口
- 中间区域显示图片浏览
- 顶部由空态 `上传` 切换成素材态通用动作栏
- 下方编辑区继续存在

顶部栏状态切换已确认：
- 空态顶部：只显示 `上传`
- 有图片内容后顶部：切换为通用动作栏

## 4.4 顶部编辑动作
当前已确认素材态通用动作栏包括：
- 重绘
- 擦除
- 增强
- 扩图
- 抠图
- 多角度
- 打光
- 标注
- 裁剪
- 下载
- 全屏

实现优先级已确认：
- 第一阶段先实现：`标注 / 裁剪 / 下载 / 全屏`
- 前 7 个主编辑动作当前先保留字段与入口，不作为第一阶段必须打通项

## 4.5 底部编辑区
当前已确认包含：
- 文本输入
- 模型位
- 参数位
- 焦点按钮
- 参考图槽位
- 标签 chips
- 积分
- 生成按钮

## 4.6 模型口径
当前产品层默认判断：
- 默认模型：`香蕉 2.0`
- 升级档：`香蕉 Pro`

## 4.7 多结果
当前已确认：
- 图片生成支持多个结果
- 主图 + 副图切换
- 下游默认引用当前主图

## 5. Video 节点

## 5.1 定位
Video 节点是：
- 文生视频节点
- 图生视频节点
- 视频生视频节点
- 视频分析节点
- 视频增强/后处理入口节点

## 5.2 模式切换
当前已确认：
- 单独创建视频节点时，默认是文生视频
- 接入图片后，可切图生视频
- 接入视频素材后，可进入视频生视频/素材态

## 5.3 左右连接规则
当前已确认：
- 左侧可接：文本 / 图片 / 音频
- 右侧输出创建菜单当前主要面向文本或视频族下游

## 5.4 素材态结构
上传视频后，节点采用三段结构：
- 顶部工具栏
- 中部视频预览
- 底部播放控制条

顶部当前已确认：
- 剪辑(beta)
- 解析
- 高清
- 替换
- 移除
- 下载
- 全屏

## 5.5 视频分析
当前已确认：
- 视频分析结果挂在当前视频节点内部
- 不默认生成独立可连线分析节点
- 当前至少支持两种浏览方式：
  - 详细分析列表视图
  - 创意图缩略图/镜头卡片视图
- 本质上是让大模型按视频内容抽帧、拆镜头、生成结构化 `shots[]` 结果集，再用两种 UI 展示同一份结构化数据
- 当前结果不能直接拆成可独立连线的镜头节点；后续增强方向才是“镜头图片节点 + 自动打组”

## 5.6 快照
当前已确认：
- 快照按当前播放/暂停时刻截图
- 自动生成新的图片节点
- 该图片节点能力与普通图片节点一致

## 5.7 高清增强
当前已确认：
- `高清` 更像派生增强能力
- 对 Veo 3.1 而言，4K 是单独后处理接口，不是普通生成分辨率下拉

## 6. Audio 节点

## 6.1 定位
Audio 节点当前至少覆盖：
- 文字转语音
- 音乐生成
- 音频素材管理
- 向视频/数字人提供音频输入

结合官方能力，后续还可扩到：
- STT
- Dialogue
- Sound Effect
- Audio Isolation

## 6.2 基础模式
当前 UI 已确认第一按钮切换：
- `文字转语音`
- `音乐`

## 6.3 素材态
上传音频后：
- 中间显示音频缩略/素材态
- 下方出现波形图
- 有播放/暂停/时间/倍速
- 顶部有 `1x` 倍速、下载
- 右上角 `上传` 语义为替换当前音频素材
- 上传后通常只保留右输出口

## 6.4 输入输出
当前已确认：
- 左侧输入只支持文本节点
- 右侧输出只支持：
  - 视频生成
  - 数字人视频

## 6.5 数字人视频
当前保守定义：
- 不作为独立节点类型处理
- 先按 Video 节点处理
- 产品层可保留 `数字人视频` 入口名
- 内部类型仍归 `video`

---

## 7. 数字人视频

当前产品判断：
- 更像 `video` 节点的一个入口/预设
- 很可能尚未真正开发成独立节点体系
- 当前不单独拆 schema，不单独做一套 UI

---

## 7.1 Group / Template / Asset

当前产品口径已确认：
- Group：保存当前选中的所有节点，以及它们当前的连接关系；本质是局部工作流单元
- Template：保存整套工作流模板，包括全部节点、全部连线、节点位置、节点配置、组信息等；本质是完整 workflow template
- Asset：保存可复用内容对象，当前主类型以图片资产为主，常见为模特/产品/角色的多角度、多景别、多特写图片集合，主要供后续图生视频等流程复用

---

## 8. 节点链接总表

### 8.1 已确认主链路
- Text -> Text
- Image -> Text
- Text -> Image
- Image -> Image
- Text -> Video
- Image -> Video
- Audio -> Video
- Audio -> 数字人视频（内部仍按 Video 处理）

### 8.2 当前重要例外
- Text 分析结果不自动流向下游
- Video 分析结果不默认变独立分析节点
- Audio 上传素材后只做素材/结果分发，不再保持完整多输入态

---

## 9. 当前最重要的产品判断

1. 节点不是纯输入框，而是复合工作单元
2. Text 节点工具条是结构化提示词注入，不是普通排版
3. Text 分析、Video 分析结果都默认挂当前节点，不自动派生成新节点
4. `插入到下层开头` 是显式写入动作，不是 edge 自动传值
5. `数字人视频` 当前按 `video` 处理，不当独立节点类型
6. 视频模型参数区必须 model-aware，不能共用一套固定表单
