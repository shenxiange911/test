## 1. 节点定位

图片节点不是静态素材卡，也不是单纯上传组件。

当前更准确的定义是：

**titanX 中用于图片生成、图片承载、参考图融合、图片编辑、结果切换、上下游编排与 AI 图片工作流触发的复合执行节点。**

它同时承担 5 类职责：
- 图片输入节点
- 图片生成节点
- 图片结果节点
- 图片编辑入口节点
- 工作流中的图片上游资产节点

---

## 2. 整体结构

图片节点当前已经可以稳定拆成三段：

### 2.1 上段
负责上传、编辑工具栏、模式切换。

典型内容：
- 上传按钮
- 上传/替换按钮
- 图片编辑动作栏
- 特定模式标题/退出按钮

### 2.2 中段
负责图片浏览、结果显示、输入/输出口、局部交互。

典型内容：
- 图片缩略预览
- 当前结果图
- 空态占位图
- 左输入口 / 右输出口
- 多图结果主图区域

### 2.3 下段
负责 prompt 输入、参考图、标签、参数、积分、提交。

典型内容：
- 大输入框
- 焦点按钮
- 参考图缩略区
- 标签 chips
- 底部参数条
- 积分 + 提交按钮

## 3. 节点状态机

图片节点当前至少存在 7 类稳定状态。

### 3.1 空态
- 没有真实图片
- 默认不展开
- 中间只显示空占位
- 可进入上传流程
- 左右口可能同时存在

### 3.2 未展开浏览态
- 节点已存在但未展开
- 只显示中间图片卡片
- 上下段不显示
- 用于快速浏览画布

### 3.3 展开编辑态
- 显示完整三段
- 上段可上传/切模式
- 中段显示当前图
- 下段显示 prompt 与参数条

### 3.4 已载入图片态
- 节点内部已有真实图片
- 节点更偏“图片资产卡”
- 默认主要承担继续输出到下游的角色
- 上传动作变成上传/替换

### 3.5 通用图片编辑态
- 顶部出现图片编辑工具栏
- 下段输入框仍然保留
- 支持多种编辑子模式

### 3.6 子模式态
当前已确认的子模式包括：
- 重绘
- 擦除
- 扩图
- 标注
- 裁剪
- 多角度
- 打光

### 3.7 下游派生态
当前已确认两类：
- 派生配置节点：增强
- 一键派生结果节点：抠图

## 4. 顶部编辑工具栏

上传图片后，顶部会出现一条独立工具栏。

### 4.1 主编辑动作
- 重绘
- 擦除
- 增强
- 扩图
- 抠图
- 多角度
- 打光

### 4.2 通用操作
- 标注
- 裁剪
- 下载
- 全屏

这两组按钮不能混为一类：
- 前一组主要是生成/编辑能力入口
- 后一组更偏查看、导出、局部工具

## 5. 左右连接口规则

图片节点左右两侧不是对称设计。

### 5.1 左侧输入口
语义：添加上下文 / 接入上游。

当前已确认：
- 只在特定展开/可接入状态下显示
- 可拖拽
- 菜单标题：`添加上下文`
- 当前仅支持两类上游：
  - 文本生成
  - 图片处理

### 5.2 右侧输出口
语义：引用当前节点继续生成下游。

当前已确认：
- 上传图片后通常只保留右侧输出口
- 右侧 `+` 支持拖拽
- 菜单标题：`引用该节点生成`
- 当前支持四类下游：
  - 文本生成
  - 图片生成
  - 视频生成
  - 图片编辑器

### 5.3 当前产品判断
- 左侧输入口能力范围小于右侧输出口
- 左侧偏接入上下文
- 右侧偏基于当前图继续生成下游

## 6. 底部输入区逻辑

图片节点底部不是单纯 prompt 框，而是一个混合输入系统。

### 6.1 自然语言输入
普通图片生成态下占位文案：
- `描述任何你想要生成的内容`

重绘态下占位文案：
- `描述你想改变什么...`

### 6.2 `/` 快捷命令系统
输入 `/` 后会弹出图片工作流快捷命令菜单。

当前已确认的示例：
- 多机位九宫格
- 电影级光影校正
- 角色三视图生成
- 画面推演 - 3秒后

规则：
- 需要当前节点至少已有一张图片
- 部分快捷项不是单纯补全 prompt，而是会直接触发图片工作流

### 6.3 `@` 图片引用系统
输入 `@` 后会弹出 `Add Reference` 菜单。

当前已确认：
- 菜单项为 `Image 1 / Image 2 / Image 3`
- 与上方参考图缩略区一一对应
- 说明底层是按参考图稳定索引进行引用，而不是自由文本 mention

这意味着：
- `@Image N` 应视为 reference binding 语法
- 后续 prompt 编译不能把它当普通字符串处理

## 7. 焦点模式

底部编辑区左上第一个按钮是 `焦点`。

### 7.1 进入方式
- 点击底部编辑区左上角第一个按钮
- 顶部进入 `焦点编辑` 模式
- 提示文案：`点击其他节点以提取元素`
- 右侧有 `退出`

### 7.2 核心逻辑
当前高可信逻辑：
- 当前图片节点进入焦点采样模式
- 用户点击其它图片节点
- 系统分析被点击图片
- 提取其中可复用元素
- 结果写回当前图片节点，而不是写回来源图

### 7.3 写回内容
当前至少有两类：
- 参考图缩略图
- 标签 chips

### 7.4 写回位置
- 位于底部编辑区左上方
- 紧挨焦点按钮区域
- 构成“参考图缩略区 + 标签区”

### 7.5 多来源支持
- 支持多次点击
- 可累计多张参考图
- 每张参考图可能贡献自己的元素标签或提示词片段

### 7.6 推荐工程抽象
```ts
interface FocusReferenceItem {
  refId: string
  sourceNodeId: string
  imageUrl: string
  extractedTags: string[]
  promptFragments: string[]
  selected?: boolean
}
```

```ts
interface ImageNodeFocusContext {
  focusMode: boolean
  references: FocusReferenceItem[]
  mergedTags: string[]
  mergedPromptFragments: string[]
}
```

## 8. 参考图系统

图片节点当前已经明确支持多参考图输入。

### 8.1 来源
- 手动上传
- 上游图片链接自动加载
- 焦点模式点击其它图片节点采样

### 8.2 表现
- 底部编辑区左上方显示多个参考图槽位
- 每张参考图可对应 `Image N`
- 可被 `@Image N` 精确引用

### 8.3 当前高可信实现判断
更合理的底层结构不是只存图片 URL，而是：
- 参考图 ID
- 来源节点 ID
- 缩略图
- 提取的元素标签
- 提取的提示词片段
- 是否启用

## 9. 底部 1~7 号参数条

当前底部参数条已经可以按按钮位顺序稳定拆开。

### 9.1 第一个按钮：图片模型类型
- 当前已确认存在模型位
- 在不同场景下至少出现过 `全能图片模型 Pro`
- 某些模式下存在 `PRO / NORMAL` 两档
- 下拉菜单当前已明确出现过的候选至少包括：
  - `全能图片模型 2`
  - `全能图片模型 Pro`
  - `全能图片模型`
  - `悠船 V7`
  - `悠梦 Niji7`
  - `即梦5.0 Lite`
  - `TapNow Flash`
  - `即梦4.0`
  - `即梦4.5`
  - `Flux 2 Pro`
  - `Flux Max`
  - `Flux`

当前判断：
- 第一个按钮不是单一模型切换，而是图片模型总入口
- 不同模型族后续很可能对应不同能力、时长和参数壳

### 9.2 第二个按钮：比例 + 画质组合控件
默认：
- 自适应当前图片比例

展开后：
- 画质：`1K / 2K / 4K`
- 比例：`自适应 / 1:1 / 9:16 / 16:9 / 3:4 / 4:3 / 3:2 / 2:3 / 5:4 / 4:5 / 21:9`

收起后显示示例：
- `自适应 · 2K`

### 9.3 第三个按钮：风格按钮
- 点击展开风格参考图
- 当前收起态显示：`风格 + Sony Veni...`
- 风格参考图支持用户自己上传
- 选中风格参考图后，会发生两类写回：
  - 风格参考图加载到当前图片节点的参考图区
  - 风格信息写入提示词编译链，供图片模型读取
- 当前高可信实现判断：风格图不是只做 UI 展示，而是在点击积分生成时，作为隐式全局提示词前缀注入到底层 prompt
- 用户在文本框里只看到自己输入的内容，看不到这部分系统自动插入的风格提示词
- 应抽象为 `styleRefs[] + activeStyleRef`

### 9.4 第四个按钮：摄像机控制
当前已确认至少包含：
- 摄像机
- 镜头
- 焦距
- 光圈
- 保存当前摄像机设置

关键点：
- 被选中的相机 preset 会写入默认提示词
- 它更像全局摄影参数，而不是一次性 UI 选项
- 当前高可信实现判断：相机参数和风格体系属于同一类“隐式全局提示词注入”机制
- 也就是说，点击积分生成时，系统会把相机 preset 对应的摄影参数自动插入到底层 prompt 前部
- 用户在文本框中只看到手写输入，不会看到这些系统自动注入的提示词片段

### 9.5 第五个按钮：联网搜索开关
当前高可信判断：
- 是 prompt 增强/检索增强开关
- 可能调用外部搜索或大模型重写提示词
- 当前不应写死实现，只先视为 `retrievalAugmentEnabled`

### 9.6 第六个按钮：张数/乘数位
当前已确认：
- `1x / 2x / 4x`

它不仅影响生成数量，还决定结果展示逻辑：
- 生成后出现主图 + 副图
- 主图右上显示张数标识
- 右侧副图展开
- 副图支持设为主图
- 支持主副图切换
- 支持下载当前副图

### 9.7 第七个按钮：积分生成按钮
- 左侧显示本次积分消耗
- 右侧圆形向上箭头为提交入口
- 示例：`18`

## 10. 多图结果管理

多图生成的结果不是简单网格，而是“主图 + 副图”的结构。

### 10.1 当前已确认逻辑
- 选择 `2x / 4x` 后会生成多张结果
- 当前主图显示在中间
- 右侧弹出副图
- 当前主图右上显示张数标识
- 多图结果在 UI 上表现为：主图在前，副图在右侧/后侧展开，后方有层叠投影感

### 10.2 当前已确认操作
- 设为主图
- 主副图切换
- 下载当前副图
- 副图被设为主图后，会替换当前主图位置；原主图退到副图位

### 10.3 当前产品判断
这套逻辑应视为图片节点的统一多结果显示规则，而不是单一模式特例。

## 11. 编辑子模式总表

### 11.1 重绘
类型：原位编辑类

结构：
- 上：蒙版工具栏
- 中：编辑画布
- 下：蒙版来源 + 尺寸 + 数量 + 模型 + 动态积分 + 提交

特点：
- 支持从设备上传蒙版 / 从画布选择
- 依赖文本描述 + 蒙版区域

### 11.2 擦除
类型：原位编辑类

特点：
- 与重绘共用交互壳子
- 核心语义是“按蒙版擦除”
- 不强调文本改写，而是去除选区内容

### 11.3 扩图
类型：原位编辑类

特点：
- 不新建节点
- 通过拖动外扩框确定扩图边界
- 底部有比例下拉与通用执行壳

### 11.4 标注
类型：原位工具类

特点：
- 类截图软件批注模式
- 具有保存按钮 `Save`
- 有画笔、矩形、文字、颜色、粗细、撤销重做

### 11.5 裁剪
类型：原位工具类

特点：
- 中间出现裁剪框
- 底部有 `宽高比` 与 `确认裁剪`
- 顶部 hover 文案为 `裁剪`

### 11.6 多角度
类型：专用控制面板类

特点：
- 左侧伪 3D 拖拽方块
- 右侧 `旋转 / 倾斜 / 缩放`
- 有 `广角镜头` 开关
- 有 `重置`
- 右下有积分 + 提交

### 11.7 打光
类型：专用控制面板类

特点：
- 左侧 3D/HDR 灯光预览
- 右侧 `亮度 / 色温 / 主光源位置 / 轮廓光`
- 积分当前看起来固定，示例为 `20`

## 12. 派生生成类动作

### 12.1 增强
类型：派生配置类

特点：
- 点击后在右侧创建新的增强节点
- 自动建立连线
- 当前增强节点默认文案：`配置参数生成增强图像`

- tab 2

### 12.2 抠图
类型：一键派生结果类

- 点击后右侧自动生成新的抠图节点

## 15. 当前最稳定的产品结论

### 15.1 图片节点是复合执行节点，不是素材卡
它承载：
- 输入
- 生成
- 编辑
- 输出
- 下游编排

### 15.2 图片节点的关键不是单个按钮，而是“模式切换”
当前真正重要的是：
- 不同子模式切换后，UI 结构和输入语义都会变
- 不能把它当一张固定表单写死

### 15.3 图片节点底部输入区是“混合输入系统”
它同时支持：
- 自然语言 prompt
- `/` 快捷命令
- `@Image N` 参考图引用
- 焦点模式写回的标签与参考图

### 15.4 多参考图与多结果图是两套不同系统
- 多参考图：用于输入与 prompt 编译
- 多结果图：用于生成结果展示与切换

### 15.5 图片节点已经具备完整工作流语义
- 左接上游上下文
- 右建下游节点
- 节点本身还能进入完整图片编辑与生成工作流

## 16. 分段实现建议

### 16.1 当前开发优先级
