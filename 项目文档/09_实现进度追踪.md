## 当前第一优先级
1. 先把 `titanX/app` 从原型壳接成最小手动闭环：节点 state 写回、手动连线、手动运行入口、结果显示
2. 再补 `customPrompting` / compiled prompt / 上游解析，让真实输入链进入当前代码
3. 然后把 Image Node 的 `kieAdapter`、payload builder、摄像机参数映射、结果持久化做实
4. 最后再抽统一动作层（仍以手动触发链路为主）

## 当前不建议过早做的事
- 暂时不为了“看起来规范”一次性拆太多目录和空壳文件
- 暂时不在 Text / Image 最小闭环未完成前铺开全部节点族实现
- 暂时不把 runtime 持久化设计成过重系统，先满足 URL 记录和追踪字段即可
- 暂时不把未经代码核实的技术栈描述写入主文档，例如提前写入 Zustand 或不存在的节点基础字段
- 暂时不把示例里的完整模板 DSL、完整 axios 客户端、完整执行队列原样照搬进当前仓库；先做最小可用闭环
- 暂时不把自动化、skill 化、agent 驱动重新拉回当前阶段主线
