## 2. 当前已落地 / 部分落地（以 `IMPLEMENTATION_REDLINES.md` 与 `09_实现进度追踪.md` 为准）
### 已实测通过
- 节点交互已改为：缩略图常驻 + 点击节点缩略图后下方显示编辑框
- 点击空白处关闭编辑器
- 节点选中高亮与编辑器展开解耦
- Image 节点编辑框已包含：模型参数 + 摄像机控制区
- 摄像机控制区字段：
  - cameraModel
  - lensModel
  - focalLength
  - aperture
- 输出侧支持鼠标附近弹出“新建下游节点”菜单
- 按类型独立业务 ID + 删除后复用最小缺失编号

### 已开工但未闭环
- 输入输出真实绑定已进入主线，但节点互联能力仍需按产品行为继续验收，不能仅以代码存在判定完成
- 图片生成已接 KIE runtime 调用链原型，但 Image Generate 真闭环（真实返回图片并显示在节点中）仍不得写成已完成

### 当前明确未完成
- `customPrompting` / 模板引用 / compiled prompt 追踪还未真正完整接入代码
- `compileNodePrompt()` 尚未成为唯一 prompt 真相来源
- 独立 `kieAdapter` / `executionPayloadBuilder()` / task 轮询标准化 / 结果持久化到 runtime records 仍未完全补齐

## 5. 清上下文后下一步先做什么
### 第一优先级
补齐 `customPrompting` / 模板引用 / compiled prompt 追踪，让它们真正进入代码，而不是只停留在文档和占位 UI。

### 第二优先级
把 Image 节点的摄像机/镜头参数真正并入 KIE 图片 payload builder，并补上独立 `kieAdapter`、标准化 task 提交/轮询与结果持久化。

### 第三优先级
把 analyze / generate / pushDownstream 收口成统一动作层；在 Text / Image 最小闭环稳定前，不要把 Video / Audio / ResultNode 大面积铺开后又回头补基础闭环。

### 并行文档修正项
把外层：
- `titanLX/严格开发文档/README.md`
改成入口/镜像说明，避免文档双轨；但这件事的优先级低于最小可执行闭环。
