# titanX IMPLEMENTATION_REDLINES

> 版本：2026-03-11 Manual-first 修正版  
> 定位：这是 titanX 当前阶段的**实现红线文档**。  
> 目的：防止开发再次滑回“旧版大表单节点壳 + 伪闭环 + 理论可连线 + 提前谈自动化”的错误路线。

## Manual-first 阶段定义
当前阶段只做**手动闭环**：
- 手动建节点
- 手动连线
- 手动编辑参数
- 手动触发 Generate / Analyze
- 真实执行
- 结果显示
- runtime 写回

**自动化暂不纳入本阶段验收**。

## 0. 一句话结论
当前阶段允许保留并优先验收的已完成能力只有：
- 新建节点
- 稳定业务 ID 分配与最小缺失编号复用（先以手动创建路径为准）
- 节点之间基础连接/删边/删节点链路
- 模型 / 比例 / 尺寸等基础参数入口

除此之外，**本轮真正未完成的核心主线**必须明确视为未完成：
1. 节点默认态仍然没有完全做成“缩略卡片态”
2. 编辑区没有稳定做到“节点下方独立展开”
3. 节点互联能力没有经过严格产品行为验收
4. 最关键的 Image Generate 真闭环未完成：**点击 Generate 后没有真实返回图片并显示在节点中**

---

## 1. 当前真正剩余目标
### 1.1 UI 形态目标
你要做的不是“可编辑的大节点”，而是：
- 节点默认态 = 缩略图 / 摘要卡片
- 编辑区 = 点击缩略卡片后，在节点下方独立展开的编辑面板

### 1.2 编排目标
你要做的不是“理论上可以连线”，而是：
- Text -> Image 真实可连
- Image -> Image 真实可连
- Image -> Text 真实可连
- 连线后上游内容真正进入下游上下文

### 1.3 运行时目标
你要做的不是“点 Generate 后状态变化”，而是：
- 点击 Generate
- 发起真实 API 请求
- 返回真实图片
- 节点中显示真实结果图
- 失败时节点中可见明确错误信息

---

## 2. 明确哪些已经做了，哪些不能再冒充“已完成”
### 2.1 可以承认已做的
- 新建节点
- 按类型分配业务 ID（如 text-01 / image-01）
- 删除节点
- 删除边
- 基础 handle / 连接规则
- 模型 / 比例 / 尺寸选择器
- 编译提示词 preview 的 UI 占位或展示入口

### 2.2 不能再冒充“已完成”的
- 节点默认态仍是大表单时，不能说“缩略图/编辑区已分离”
- 只有代码里有 handle / connectionRules 时，不能说“节点互联已完成”
- 只有 payload / adapter / writeback 代码存在时，不能说“真实生成闭环已完成”
- 只有 success/error 状态变化时，不能说“生成结果返回成功”

---

## 3. 一票否决项
以下任一项不满足，本轮直接判失败：

### 3.1 默认态仍是长表单节点
如果节点默认看到的仍然是完整表单、参数面板、长编辑器，而不是紧凑卡片态，直接失败。

### 3.2 编辑区未与缩略卡片明确分离
如果编辑区只是节点主体继续向下拉长，而不是“节点下方独立展开面板”，直接失败。

### 3.3 节点之间任一核心方向无法真实互联
以下任一实际不能连：
- Text -> Image
- Image -> Image
- Image -> Text
直接失败。

### 3.4 Generate 不出图
如果点击 Generate 后：
- 没有真实 API 结果图
- 节点中没有显示返回图片
- 只有状态变化没有图
直接失败。

### 3.5 只做代码结构、不做产品行为验证
如果结论只是“代码存在 / 结构合理 / 理论支持”，没有产品行为证据，直接失败。

## 4. 错误示范（禁止继续实现）
### 错误示范 A：大表单塞进节点主体
错误：默认节点就是完整参数表单。  
正确：默认节点只显示卡片摘要，点击后下方展开编辑区。

### 错误示范 B：节点内部多露几行内容就算编辑区分离
错误：同一个卡片内展开更多字段。  
正确：下方独立编辑区，视觉上与卡片明确分层。

### 错误示范 C：写了连线规则就算互联完成
错误：只证明代码里存在 handle / edge 逻辑。  
正确：三种核心连接方向必须实际可连、可删除、可传递上游数据。

### 错误示范 D：Generate 只返回 success/error 就算闭环
错误：状态 badge 变化但没有结果图。  
正确：节点中必须显示真实返回图片。

---

## 5. 当前阶段验收最小闭环
当前阶段真正需要验收的，不是抽象架构，而是以下最小闭环：
1. 新建 TextNode
2. 新建 ImageNode
3. Text -> Image 连线
4. 点击缩略卡片展开下方编辑区
5. ImageNode 读取上游文本
6. 配置模型 / 比例 / 尺寸
7. 点击 Generate
8. 返回真实图片
9. 节点中显示结果图
10. 删除节点 / 删除边后结构仍正确

## 6. 报告书写红线
后续所有报告必须明确区分：
- 已实测通过
- 仅代码/文件证据通过
- 未验证

禁止把以下内容写成 PASS：
- “代码里有”
- “理论上支持”
- “组件存在”
- “应该可用”
- “自动化以后可以接上”

只有当前阶段**手动实际行为通过**，才能写 PASS。
