记录时间：2026-03-11

---

## 1. 设计原则

统一 schema 不等于所有模型都共用同一套表单。
统一 schema 的正确理解是：
- 所有节点共享统一外壳
- 各节点有自己的内容层和编辑层
- 运行时和结果结构尽量统一
- 模型参数必须允许按模型家族动态变化

---

## 2. 通用节点外壳

```ts
interface BaseNodeShell {
  nodeId: string
  businessId: string
  nodeType: 'text' | 'image' | 'video' | 'audio'
  title?: string
  displayLabel?: string
  collapsed: boolean
  selected: boolean
  status: 'idle' | 'editing' | 'running' | 'success' | 'error'
  hasLeftInput: boolean
  hasRightOutput: boolean
  position?: { x: number; y: number }
  tags?: string[]
}
```

说明：
- `businessId` 必须按类型独立编号，例如 `text-01`、`image-01`
- 删除后应复用最小缺失编号

## 5. Text 节点

```ts
interface TextNodeData extends BaseNodeShell {
  nodeType: 'text'
  content: string
  editorPrompt?: string
  inputRefs?: Array<{ refType: 'text' | 'image'; nodeId: string }>
  analysisMode?: 'manual-prompt-analysis'
  nodeColor?: string
  toolbarSemanticBlocks?: Array<'h1' | 'h2' | 'h3' | 'paragraph' | 'bold' | 'italic' | 'bullet-list' | 'ordered-list' | 'divider'>
  downstreamWriteback?: {
    actionLabel: '插入到下层开头'
    enabledTargetTypes: Array<'text' | 'image' | 'video' | 'audio'>
    behavior: 'prepend'
  }
  runtime?: CommonRuntimeState
  resultState?: MultiResultState
}
```

关键规则：
- `content` 是中间正文区
- `editorPrompt` 是底部执行输入
- 分析结果写回 `content`
- `inputRefs` 只表示挂载上游，不表示自动执行
- `nodeColor` 只影响当前 Text 节点外观，不进入 prompt 注入链
- 进入 prompt 编译链的是 `toolbarSemanticBlocks` 中从 `h1` 到 `divider` 的结构化语义块

  nodeType: 'video'
  mode?: 'text-to-video' | 'image-to-video' | 'video-to-video'

  inputImages?: string[]
  inputVideos?: string[]
  inputAudios?: string[]

  duration?: string

  multiShots?: boolean

  resultState?: MultiResultState

关键规则：
- `analysis` 挂当前视频节点
- 不默认生成独立分析节点
- `subtype='digital-human'` 当前仅作产品预设标记

## 10. 当前最重要的 schema 结论

1. 统一 schema 的重点是统一外壳和 runtime，不是统一所有模型参数
2. Text / Video 的分析结果都默认挂当前节点内部
3. Video / Audio 的产品入口可有独立名字，但内部类型和 subtype 要分开
4. 视频模型参数必须放到 model-aware schema 层，不能硬写死在 `VideoNodeData` 顶层常量表单里
