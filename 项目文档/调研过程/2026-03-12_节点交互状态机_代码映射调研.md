`/home/liudehua/.openclaw/workspace/titanX/app/src/App.jsx` 的 `ReactFlow onPaneClick`

因为这个动作本质上是 React Flow pane 级事件，不是右侧面板事件。

## 7. picking mode 应该挂在哪层

### 7.1 当前最适合挂在 `App.jsx` 的画布状态层
原因：
- picking mode 会跨节点
- picking mode 要影响整个画布点击语义
- 它不是单个节点自己的局部状态

### 7.2 当前代码还缺的状态
建议在 `App.jsx` 或后续 `flowStore` 增加：
```js
canvasMode
focusPickState
```

例如：
```js
canvasMode = 'idle' | 'focus-pick' | 'creating-from-canvas'
focusPickState = {
  active: false,
  sourceNodeId: null,
  targetKind: null,
}
```

### 7.3 如何避免污染普通选中逻辑
关键规则：
- `onNodeClick` 先判断 `canvasMode`
- 如果 `canvasMode === 'focus-pick'`，就不走普通 `setSelected(node)`
- 而是走 `handleFocusPick(node)`

也就是说：
- picking mode 是对 `onNodeClick` 的“分流”
- 不是把 `selected` 改造成一个混合态

当前代码映射：
