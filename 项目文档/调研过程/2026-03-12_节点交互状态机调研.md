## 2. 核心交互规则提取

### 2.1 缩略图常驻规则
从 `[ANCHOR-UI-001]` 提取：
- 节点不是点击展开整个节点本体
- 节点必须保持缩略图/摘要卡片常驻
- 只有点击节点缩略图时，才显示下方编辑框

**状态机含义**：
- 节点有两个独立状态维度：
  - `selected`：节点是否被选中（高亮边框）
  - `editorExpanded`：下方编辑器是否展开
- 这两个状态必须解耦

### 2.2 选中高亮与编辑器展开解耦
从 `[ANCHOR-UI-001]` 提取：
- 节点选中高亮与编辑器展开必须解耦

**状态机含义**：
- 可以选中节点但不展开编辑器（例如框选、准备移动）
- 可以展开编辑器但不选中节点（理论上，但产品可能不允许）
- 实际产品语义：点击缩略图 → 选中 + 展开编辑器

### 2.3 点击画布空白关闭编辑器
从 `[ANCHOR-UI-001]` 提取：
- 点击画布空白处时，应关闭当前编辑器

**状态机含义**：
- 画布空白点击事件应触发全局 `closeAllEditors()`
- 但不一定清空 `selected` 状态（取决于产品是否允许保持选中但关闭编辑器）

### 2.4 上传素材后切素材态
从 `[ANCHOR-VIDEO-001]` 和 `[ANCHOR-AUDIO-001]` 提取：
- 上传视频后，节点进入素材态
- 上传音频后，节点进入素材态
- 素材态特征：
  - 只保留输出口，不保留左输入口
  - 顶部出现素材操作工具栏
  - 中部是素材浏览器
  - 底部是播放控制条

**状态机含义**：
- 节点有 `mode` 状态：
  - `empty`：空节点
  - `editing`：编辑态
  - `asset`：素材态
  - `result`：结果态
- `mode` 切换会影响：
  - 输入输出口可见性
  - 顶部工具栏内容
  - 中部显示内容
  - 底部控制区内容

### 2.5 多结果主次切换
从 `[ANCHOR-UI-001]` 和 `[ANCHOR-UI-005]` 提取：
- 多结果生成后必须支持主次切换
- 后续下载、创建资产、继续连接下游时，默认作用于当前主结果
- 多图生成结果必须按"主图 + 右侧副图展开"展示
- 副图必须支持：设为主图、主副图切换、下载当前副图

**状态机含义**：
- 节点结果不是单一值，而是 `MultiResultState`：
  - `primaryResultId`
  - `results[]`
- 切换主结果不改变节点整体状态，只改变 `primaryResultId`

### 2.6 picking mode 进入/退出
从 `[ANCHOR-VIDEO-002A]` 提取：
- 进入焦点编辑后，画布进入 picking mode
- 提示为"点击其他节点以提取元素"
- 带 `退出` 按钮

**状态机含义**：
- 这是画布级状态，不是单节点状态
- 画布状态：`canvasMode`
  - `idle`
  - `focus-pick`
  - `creating-from-output`
  - `grouping`
- 当 `canvasMode = 'focus-pick'` 时：
  - 当前节点进入 `waitingForFocusPick` 状态
  - 其他节点变为可点击提取态
  - 画布显示退出按钮

### 2.7 节点选中、编辑、运行、结果态切换
从 `BaseNodeShell` 和 `CommonRuntimeState` 提取：
- 节点有 `status`：
  - `idle`
  - `editing`
  - `running`
  - `success`
  - `error`
- 节点有 `runtime.state`：
  - `waiting`
  - `success`
  - `fail`

**状态机含义**：
- `status` 是节点整体状态
- `runtime.state` 是运行时状态
- 两者关系：
  - `status = 'running'` 时，`runtime.state = 'waiting'`
  - `status = 'success'` 时，`runtime.state = 'success'`
  - `status = 'error'` 时，`runtime.state = 'fail'`

## 3. 四类节点共性分析

### 3.1 Text 节点特殊性
- 没有素材态（不支持上传文本文件后进入素材浏览态）
- 分析结果写回 `content`，不是独立结果态
- 有 `插入到下层开头` 特殊动作

### 3.2 Image 节点特殊性
- 有 `mode`：`generate` / `edit` / `asset`
- 顶部有 7 个编辑动作按钮（当前只部分实现）
- 有摄像机参数区
- 有风格参考图区
- 多结果展示为主图 + 副图

### 3.3 Video 节点特殊性
- 有素材态（上传视频后）
- 有 `mode`：`text-to-video` / `image-to-video` / `video-to-video`
- 有 `subtype`：`digital-human` / `video-enhance`
- 有解析结果内嵌
- 有快照派生、高清派生能力
- 有剪辑(beta)能力

### 3.4 Audio 节点特殊性
- 有素材态（上传音频后）
- 有 `mode`：`tts` / `music` / `dialogue` / `stt` / `sfx` / `audio-isolation`
- 有播放器控制
- 有波形图显示

### 3.5 共性总结
所有四类节点都有：
- 缩略图常驻
- 点击缩略图展开编辑器
- 选中高亮
- 运行态
- 结果态
- 多结果支持（除 Text）
- 输入输出口
- 右键菜单
- 可打组
- 可创建资产

## 4. 状态维度拆分

### 4.1 节点级状态（存 React Flow node data）
- `nodeId`：React Flow id
- `businessId`：业务 ID（如 `text-01`）
- `nodeType`：`text` / `image` / `video` / `audio`
- `mode`：节点模式（各类型不同）
- `status`：`idle` / `editing` / `running` / `success` / `error`
- `collapsed`：是否折叠（当前产品未用）
- `hasLeftInput`：是否有左输入口
- `hasRightOutput`：是否有右输出口
- `runtime`：运行时状态
- `resultState`：结果状态

### 4.2 UI 级状态（存外层 UI state）
- `selected`：是否选中
- `editorExpanded`：编辑器是否展开
- `editorNodeId`：当前展开编辑器的节点 ID
- `focusPickingSourceNodeId`：焦点拾取源节点 ID
- `multiSelectionIds`：多选节点 ID 列表
- `groupId`：所属组 ID

### 4.3 画布级状态（存 flowStore）
- `canvasMode`：`idle` / `focus-pick` / `creating-from-output` / `grouping`
- `contextMenu`：右键菜单状态
- `creationMenu`：创建菜单状态
- `pendingConnection`：待建连接状态

## 5. 状态转换规则

### 5.1 缩略图点击
```
触发：点击节点缩略图
前置条件：节点存在
状态变化：
  - selected = true
  - editorExpanded = true
  - editorNodeId = 当前节点 ID
  - 其他节点的 editorExpanded = false
```

### 5.2 画布空白点击
```
触发：点击画布空白处
前置条件：无
状态变化：
  - editorExpanded = false（所有节点）
  - editorNodeId = null
  - selected = false（可选，取决于产品）
```

### 5.3 上传素材
```
触发：上传文件到节点
前置条件：节点类型支持素材上传
状态变化：
  - mode = 'asset'
  - hasLeftInput = false
  - hasRightOutput = true
  - status = 'success'（素材加载成功）
```

### 5.4 生成执行
```
触发：点击生成按钮
前置条件：节点参数就绪
状态变化：
  - status = 'running'
  - runtime.state = 'waiting'
  - runtime.taskId = 新任务 ID
```

### 5.5 生成成功
```
触发：API 返回成功
前置条件：status = 'running'
状态变化：
  - status = 'success'
  - runtime.state = 'success'
  - resultState.results = 新结果列表
  - resultState.primaryResultId = 第一个结果 ID
```

### 5.6 生成失败
```
触发：API 返回失败
前置条件：status = 'running'
状态变化：
  - status = 'error'
  - runtime.state = 'fail'
  - runtime.failCode = 错误码
  - runtime.failMsg = 错误信息
```

### 5.7 主结果切换
```
触发：点击副图"设为主图"
前置条件：resultState.results.length > 1
状态变化：
  - resultState.primaryResultId = 新主图 ID
  - 不改变 status
```

### 5.8 焦点拾取进入
```
触发：点击焦点按钮
前置条件：节点支持焦点拾取
状态变化：
  - canvasMode = 'focus-pick'
  - focusPickingSourceNodeId = 当前节点 ID
  - 画布显示退出按钮
```

### 5.9 焦点拾取退出
```
触发：点击退出按钮 或 完成拾取
前置条件：canvasMode = 'focus-pick'
状态变化：
  - canvasMode = 'idle'
  - focusPickingSourceNodeId = null
  - 画布隐藏退出按钮
```

## 7. 不建议 v1 过早实现的复杂态

### 7.1 视频解析结果自动拆镜头节点
理由：
- 当前产品确认解析结果先按内嵌结果框落地
- 自动拆镜头涉及复杂的节点派生 + 自动打组逻辑
- 建议先做通解析结果展示 + 手动复制

### 7.2 组布局自动排版
理由：
- 当前产品确认有宫格布局、水平布局
- 但自动排版算法复杂
- 建议先做通手动打组 + 整组移动

### 7.3 整组执行的完整 runtime 写回
理由：
- 整组执行涉及多节点并发/串行执行
- 涉及依赖解析、错误传播、部分成功处理
- 建议先做通单节点执行 + 整组执行入口

### 7.4 焦点拾取的复杂写回逻辑
理由：
- 当前产品确认焦点拾取会写回参考图 + 标签
- 但写回逻辑涉及多种节点类型的适配
- 建议先做通基础焦点拾取 + 简单写回

## 9. 调研结论

### 9.1 核心发现
1. 节点状态必须拆成三层：节点级、UI 级、画布级
2. `selected` 和 `editorExpanded` 必须解耦
3. `mode` 和 `status` 是两个独立维度
4. 素材态是特殊的 `mode`，不是 `status`
5. 多结果是节点内部状态，不是多个节点

### 9.2 最重要的实现建议
1. 不要把所有状态都塞进 React Flow node data
2. UI 状态（如 `editorExpanded`）应该放外层
3. 画布状态（如 `canvasMode`）应该放 flowStore
4. 节点状态（如 `mode` / `status` / `runtime`）才放 node data

### 9.3 v1 最小实现路径
1. 先做通缩略图点击 → 展开编辑器
2. 再做通画布空白点击 → 关闭编辑器
3. 再做通生成执行 → 运行态 → 结果态
4. 再做通多结果主次切换
5. 最后做通素材态 + 焦点拾取
