## 1. 模块名称
中间功能

## 2. 模块定位
该模块指 titanX 画布中部的核心编排区与空画布起手区，负责：
- 展示节点之间的关系与生成链路
- 作为用户添加新节点、连接节点、开始创作的主要区域
- 在空白状态下给出快速起手入口
- 在已有节点状态下承载 Text / Image / Video 等节点的串联

它本质上是整个工作流的“中央操作区 / 中央画布区”。

### 3.2 已有节点时的中间编排功能
当画布中已有节点后，中间区域支持：
- 展示节点卡片
- 展示节点间连线
- 支持从上游节点流向下游节点
- 支持至少以下链路示例：
  - Image -> Video
  - Image -> Image
  - Text -> Video

### 3.3 节点展示形态
从你发来的截图看，中间区域当前节点以卡片形式展示，至少包含：
- Text 节点卡片
- Image 节点卡片
- Video 节点卡片

节点表现特征：
- Image 节点可直接显示缩略图
- Text 节点可直接显示文本内容摘要
- Video 节点在未生成/未加载结果时显示占位区域
- 连线以可视化方式表达上下游关系

### 3.4 中间模块中的“我的模板”子功能

- 左侧分组区：
  - 最近使用
  - 我的模板

- 公开分类下当前可见项：
  - 全部
  - 全部图片模型2
  - 广告
  - 电商
  - 影视
  - 生活
  - 工具
  - 作图

  - 搜索框（用于搜索模板）
  - 搜索按钮
  - 模板列表区
  - 空状态提示：没有更多数据

这个子功能说明：
- 中间模块支持“从模板开始”而不只是“从空白开始”
- 模板系统至少区分“最近使用 / 我的模板 / 公开模板”三层来源
- 公开模板支持按分类浏览
- 模板面板具备搜索能力

### 3.5 中间区域的工作流含义
当前模块体现的是“节点式工作流编排”：
- 上游节点产出结果
- 结果可作为下游节点输入或参考
- 通过连线表达依赖关系
- 允许不同模态之间串联

## 4. 从截图推定出的模块职责
以下为基于截图可稳定确认的职责，不含超出截图的过度推断：

1. **作为主画布**
   - 承载节点摆放
   - 承载节点之间关系展示

2. **作为创建入口**
   - 空白状态下给出快速创建选项
   - 降低首次使用门槛

3. **作为链路编排入口**
   - 支持多节点顺序组织
   - 支持至少文本、图片、视频三类节点参与链路

4. **作为结果承载区**
   - 节点卡片本身可直接显示内容或结果预览

## 6. 当前已补充确认的画布交互规则
你后面补充后，这些交互已经可以先记为中间画布的正式功能：

- **空白画布双击新建**：一开始双击画布可新建 `文本`、`图片`、`视频`、`音频`、`图片编辑器`
- **上传附件自动识别**：上传附件后，系统会自动识别并归入对应节点类型
- **上传入口支持自适应创建节点**：左侧添加节点面板底部的 `上传` 入口当前明确支持图片、视频、音频、文本/TXT 文件；上传后不是落到统一素材池，而是按文件类型直接自适应创建对应节点
- **框选多个节点**：按住鼠标拖动可框选多个节点
- **右键拖动画布**：按住鼠标右键可拖动画布视图
- **右键菜单**：当前已确认右键菜单包含：
  - 上传
  - 添加资产
  - 添加节点
  - 撤消
  - 重做
  - 粘贴

这些说明中间画布不仅是展示层，而是完整的编辑器交互层。

## 6.1 单节点右键菜单
你这次补充后，单个节点被选中时的右键能力也可以先锁定：
- 编辑
- 创建资产
- 复制
- 粘贴
- 副本
- 删除
- 复制到剪贴板

这说明单节点右键菜单不仅是基础编辑菜单，还承担：
- 节点转资产
- 节点复制派生
- 节点结果外部复制

## 6.2 多选后的批量能力
框选多个节点后，当前可确认：
- 可对框选结果执行 `创建资产`
- 可 `添加到现有资产`

这次补图后，这两项也可以更具体：
- `创建资产` 会打开独立面板，当前至少包含：
  - 封面预览
  - 名称输入
  - 分类下拉
  - `创建` 按钮
- `添加到现有资产` 会打开独立面板，当前至少包含：
  - `创建资产 / 添加到现有资产` tab 切换
  - 分类筛选：`全部 / 人物 / 场景 / 物品 / 风格 / 音效 / 其他`
  - 结果列表区
  - `添加` 按钮

这说明框选不是纯视觉选择，而是批量资产化入口。

## 6.3 打组规则
你这次补充后，画布上的“打组”已经可以先作为独立功能记下：
- 框选多个节点后支持打组
- 打组后与普通单节点存在明确差异：
  - 组颜色
  - 组布局
  - 组执行工作流
- 当前组颜色通过顶部最左侧颜色圆点控制，点击后会弹出背景颜色面板。
- 当前组布局入口已确认至少两种布局方式：
  - `宫格布局`
  - `水平布局`
- 打组后的顶部快捷栏当前已确认至少包括：
  - `整组执行`
  - `创建模板`
  - `解组`
- `整组执行` 已确认是真执行入口，不是装饰按钮
- 当前截图已确认：如果组内包含未就绪节点，整组执行会失败，并弹出失败提示
- 打组支持创建模板，本质上是把该组保存为工作流模板
- 解开打组后，组内节点恢复为独立单个节点
- 打组后仍然可以操作组里面的单个节点
- 选中组框空白处时，可以整组移动

这说明“组”不是纯视觉容器，而是：
- 工作流逻辑单元
- 模板保存单元
- 批量移动单元
- 可执行单元

## 7. 当前新增确认 · 视频节点在中间画布中的行为
你这轮补的“视频节点”已经可以作为中间画布规则的一部分补进来：

### 7.1 视频节点创建入口
当前已确认视频节点可通过以下方式进入画布：
- 左侧栏创建
- 双击画布空白处创建
- 从图片节点输出口拖拽创建
- 其他节点输出创建菜单中选择 `视频`

### 7.2 视频节点上传素材后的画布形态
当前已确认：
- 上传视频后会出现顶部工具栏
- 上传视频后当前节点只保留输出口
- 说明视频素材态更接近“结果浏览态 + 下游分发态”

### 7.3 视频节点在画布中的派生行为
当前已确认视频节点存在两类关键派生：
- `快照`：按当前播放或暂停帧截图，生成新的图片节点
- `高清`：自动创建并连接一个视频增强节点

这两条很重要，因为说明中间画布不仅支持“节点生成结果”，还支持：
- 从节点结果继续派生新节点
- 派生节点时自动建边
- 一个节点内部动作直接转成新的工作流分支
