## 1. 节点创建入口
当前已确认，音频节点至少可通过以下方式创建：
- 左侧栏创建
- 双击画布空白处创建
- 其他节点输出创建菜单中选择 `音频`
- 上传本地音频后自动识别并落到音频节点

这说明音频节点也是画布级一等节点类型，不是仅服务于视频节点的附属面板。

---

## 2. 节点基础形态

### 2.1 三段式结构
音频节点已确认继承当前 titanX 节点统一三段式：
1. 上方：上传栏 / 素材操作栏
2. 中间：音频缩略浏览区
3. 下方：独立展开编辑区

也就是说：
- 闭合态时，主要保留中间音频缩略图浏览区
- 展开态时，下面出现完整编辑区
- 文本框放大后，也只是底部编辑区的放大版，不会改变按钮结构

### 2.2 空态
当前已确认：
- 音频节点可先以空态创建
- 空态时中间显示音频占位缩略图
- 上方存在上传入口
- 左右两侧默认保留输入/输出创建口

### 2.3 上传本地音频后的素材态
用户已明确：
- 上传本地音频后，中间仍是音频缩略浏览区
- 下方会出现音频波形图
- 下方会出现播放 / 暂停、音频时长、参数控制
- 上方可调整播放速度，默认 `1x`，最高 `2x`
- 倍速按钮旁边是下载按钮
- 右上角有上传 / 替换当前音频素材按钮，逻辑与图片节点替换素材一致
- 一旦上传了音频素材，节点只保留输出口，不再保留左输入口

这说明上传音频后的节点更接近“结果浏览态 / 素材态”，而不是继续等待多源输入的编辑态。

## 3. 上传音频后的素材态结构

### 3.1 顶部操作区
当前已确认顶部至少包含：
- 播放速度调节，默认 `1x`，最大 `2x`
- 下载按钮
- 上传 / 替换当前素材按钮

其中：
- 顶部 `1x` 与下载按钮位于节点上方悬浮区
- 右上角 `上传` 按钮用于替换当前音频素材，语义与图片/视频节点替换素材一致
- 播放速度是播放器级查看控制，不是生成参数
- 替换当前素材的语义与图片/视频节点一致，即替换当前节点绑定的素材文件

### 3.2 中间浏览区
当前已确认中间区域用于承载：
- 音频缩略图 / 占位图
- 音频素材主预览
- 输出创建口

上传素材后：
- 左输入口消失
- 右输出口保留
- 该输出口继续承担把当前音频结果送往下游的职责

### 3.3 底部播放器区
上传音频后的底部当前已确认至少包含：
- 波形图
- 播放 / 暂停按钮
- 当前播放时间
- 总时长
- 相关播放参数

这里的参数更偏向播放器控制，而不是生成参数面板。

## 4. 底部编辑区：文本生成音频

### 4.1 基础骨架
当前已确认，音频节点下方独立编辑区的基础结构与前面图片 / 视频节点一致，至少包含：
- 文本输入框
- 右上角文本框放大按钮
- 字数统计
- 模式切换按钮
- 模型选择按钮
- 第三个参数 / 展开按钮
- 积分显示
- 生成按钮

并且：
- 放大后只是同一套文本框的放大版
- 下方按钮结构与缩小时一致
- 积分和生成按钮的位置逻辑与其他生成型节点保持一致

### 4.2 第一按钮：模式切换
当前已确认第一个按钮用于切换：
- `文字转语音`
- `音乐`

这说明音频节点内部至少包含两类生成模式：
- Text-to-Speech（TTS）
- Text-to-Music（TTM）

### 4.3 第二按钮：模型选择
当前已确认：
- 选择 `音乐` 模式后，第二个按钮当前只有一个音乐模型
- 当前截图可见模型为 `ElevenLabs Music`
- 该模型当前带有说明文案：`AI 音乐生成，支持多种风格和情绪`
- 当前模型卡右侧可见时长标记 `30s`
- 后期可以继续补更多音乐模型
- 选择 `文字转语音` 后，当前有两个模型
- 当前截图至少可见 `ElevenLabs V2` / `ElevenLabs V3`
- 两个模型卡右侧当前都可见 `30 s` 标记
- `ElevenLabs V3` 当前带有说明文案：`最新一代语音合成，支持情绪控制`
- 文字转语音模型后期也可能继续扩展

这说明音频节点的模型位也是 `mode-aware` 的：
- 不同模式下，可选模型集合不同
- 同一模式下，不同模型又会进一步影响下方参数区结构
- 模型下拉不仅是名称列表，还可能承载能力说明与默认时长标记

### 4.4 第三个按钮：动态参数展开位
当前已确认第三个按钮不是固定单一字段，而是模式 / 模型联动展开位。

#### 4.4.1 在音乐模式下
当前已确认第三个按钮展开后至少包含：
- `纯音乐` 开关
- `音乐时长` 选择

音乐时长当前已确认可选：
- `Auto`
- `30s`
- `1m`
- `2m`
- `3m`
- `4m`

并且：
- `Auto` 表示根据文本自动推断时长
- 音乐模式当前更像是背景音乐 / 配乐生成模式

#### 4.4.2 在文字转语音模式下
当前已确认第三个按钮展开后，右侧会带出更完整的音色与语音控制区。

至少已确认包含：
- 音色选择
- 稳定性选择 / 稳定性滑块
- 语速滑块
- 相似度滑块
- 风格强度滑块
- 右侧详细音色列表

其中：
- 左侧主参数区当前至少可见一行 `音色` 下拉框
- `音色` 下拉当前已观察到可直接显示已选音色，例如：`Roger - Laid-Back, Casual, Resonant`
- `稳定性` 当前截图里已出现三段式语义区间：`创造性 / 自然 / 稳定`
- 右侧是可滚动的详细音色列表
- 不同音色项当前至少会显示：
  - 名称
  - 一段风格描述
  - 标签摘要（如性别 / 年龄 / 气质 / 口音）
- 当前截图中已能确认的示例音色包括：
  - `Roger - Laid-Back, Casual, Resonant`
  - `Sarah - Mature, Reassuring, Confident`
  - `Laura - Enthusiast, Quirky Attitude`
  - `Charlie - Deep, Confident, Energetic`
  - `George - Warm, Captivating Storytel...`
  - `Callum - Husky Trickster - character...`
  - `River - Relaxed, Neutral, Informative...`
- 这些滑块与音色项应视为与当前 TTS 模型强相关的参数，不应强行抽成全模型共用常量面板

## 5. 输入输出规则

### 5.1 左侧输入口
当前已确认：
- 音频节点左侧 `+` 只支持文本节点输入

更准确理解是：
- 音频生成态只允许把文本作为上游输入
- 不支持从图片 / 视频直接接入音频节点左侧

### 5.2 右侧输出口
当前已确认：
- 音频节点右侧输出 `+` 只支持：
  - `视频生成`
  - `数字人视频`
- 这一点在空态和上传素材后的结果态都保持一致

说明：
- 音频节点当前主要是为视频生成链与数字人链提供音频输入
- 不支持直接从音频节点右侧继续创建图片节点或普通文本分析节点
- 右侧输出菜单的文案当前更接近“目标视频工作流类型”，而不是泛写成节点技术类型

### 5.3 上传素材后的链接变化
当前已确认：
- 一旦上传本地音频，节点只保留输出口
- 说明上传素材后的音频节点被理解为“成品音频源 / 资产源”
- 此时它主要承担向视频 / 数字人下游分发素材的职责

## 6. 数字人节点的当前产品判断

### 6.1 数字人不是独立底座，而更像视频节点的特化模式
你这次的关键判断很明确：
- 数字人节点实际上就是 `文生视频 + 加载了音频`
- 自动加载 `Wan 2.6` 模型
- 这个模型可能支持音频驱动口型或人物说话视频生成

所以当前更准确的产品理解是：
- `数字人` 不是完全独立的新运行时
- 而是视频节点家族中的一个预设分支 / 特化模式
- 本质仍然属于视频模型范畴

### 6.2 当前可先锁定的数字人规则
- 当从音频节点右侧创建 `数字人视频` 时，本质上仍是创建视频模型节点
- 默认进入 `文生视频` 形态，但同时自动载入当前音频素材
- 默认自动选择 `Wan 2.6`
- 其他参数基本继承普通视频模型
- `Wan 2.6` 当前已确认会多一个 `高级设置` 入口
- `高级设置` 当前至少关联 `多镜头` 开关
- 这个高级设置也可能承担：
  - 音频驱动口型
  - 基于音频生成角色视频
  - 多镜头参与能力

### 6.3 当前阶段的工程含义
所以当前开发时更稳妥的做法是：
- 不把数字人先做成完全独立节点类型
- 先视作 `Video Node` 的 `digital-human preset`
- 由音频节点输出创建时自动套用：
  - 默认模型 `Wan 2.6`
  - 默认载入音频输入
  - 默认进入文生视频 / 音频驱动视频语义

## 8. 本轮确认结论
当前可先把音频节点锁成以下版本：

1. 音频节点继承统一三段式结构：上方上传 / 操作，中间音频缩略图，下面独立编辑区
2. 上传本地音频后，会出现波形图、播放 / 暂停、时间信息、倍速、下载、替换素材等播放器能力
3. 上传音频后当前节点只保留输出口，不再保留左输入口
4. 音频节点左输入只支持文本；右输出只支持视频与数字人
5. 底部编辑区第一按钮切换 `文字转语音 / 音乐`
6. `音乐` 模式当前只有一个音乐模型，并带 `纯音乐` 开关与 `Auto / 30s / 1m / 2m / 3m / 4m` 时长位
7. `文字转语音` 模式当前至少有两个模型，并带音色选择、语速、稳定性、相似度、风格强度等模型相关参数
8. 文本框放大只是同一编辑区的放大版，按钮结构、积分、生成按钮逻辑与其他节点一致
9. 数字人当前更准确地理解为视频模型分支：从音频节点创建时自动套用 `Wan 2.6 + 音频输入`
